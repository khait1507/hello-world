Project Reflection: Chada Tech Dual Clock

The goal of this project was to build a dual-clock display for Chada Tech's website, allowing clients to view both a 12-hour and a 24-hour clock simultaneously. 
Rather than relying on a standard single-format clock, the program needed to accept user input and update both clocks in real time using secure and efficient C++ code.

One area where I felt I performed well was in my planning and incremental approach to development. Before writing a single line of code, I mapped out the logic in my notes so I had a clear picture of what I was trying to build. 
From there, I coded in stages rather than attempting to write everything at once. This strategy paid off because it kept bugs isolated and manageable — catching problems early is far less overwhelming than debugging an entire finished program at once.

Looking back, there are meaningful improvements I could make. Replacing certain `if` statement chains with `switch` statements where appropriate would make the control flow cleaner and easier to follow. 
I also noticed the overall length of the code could be reduced by identifying repeated logic and consolidating it into shared functions. These changes would improve readability and reduce the risk of introducing inconsistencies when the code needs to be updated later. 
From a security standpoint, tightening input validation would also make the program more robust against unexpected user entries.

The most challenging aspect of this project — and honestly this entire course — was simply the learning curve of C++. 
I leaned heavily on YouTube tutorials, Google, and repeated readings of the ZyBooks material just to keep pace. That said, I've come to accept that continuous learning isn't a weakness in this field; it's the nature of the profession. 
Every resource I used has become part of my ongoing support network, and I've gotten better at knowing where to look when I'm stuck.

The most transferable skill I developed through this project is a stronger practical understanding of functions — how to define them, how to pass data between them, and how to design a program where multiple functions work together toward a single goal. 
Functions are foundational to virtually every programming language and paradigm, so that understanding will carry directly into future coursework and projects.

To keep the program maintainable, readable, and adaptable, I followed standard C++ coding conventions as closely as I could — meaningful variable names, consistent formatting, and avoiding redundant code. 
The goal was to write something that another developer (or future me) could pick up and understand without needing extra explanation. Clean code isn't just about aesthetics; it's what makes a program practical to build on over time.
